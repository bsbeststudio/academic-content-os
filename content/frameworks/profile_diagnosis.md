# Profile Diagnosis Framework

## 定位

本框架用于在申请流程启动前，系统性诊断候选人的当前 profile 状态，输出"优先级 + 策略 + 时间线"三合一方案。

执行时机：Profile Diagnosis 是 Academic Career OS Stage 3 的核心动作，在冲刺申请前完成。

---

## 诊断前置问题

不要问"我的背景怎么样"，而要问：

1. **面试官拿到你的材料，第一反应是什么？**（Signal Inventory 视角）
2. **如果把你放进同一批次候选人，你的差异化是什么？**（Positioning 视角）
3. **如果最强的一封推荐信明天不能用了，你还能申吗？**（Constraint Check 视角）

---

## 五维诊断

### Dimension 1：Signal Inventory（信号清点）

| 信号类型 | 内容 | 当前状态（空/弱/可用/强） |
|---|---|---|
| 研究经历 | RA/助研/独立研究 | |
| 推荐信 | 推荐人及其权重 | |
| 技术能力 | Coding 能力，可展示程度 | |
| 学业表现 | GPA / 核心课程成绩 | |
| 发表/会议 | 论文/Working Paper/Poster | |

评估标准：信号须"可叙述、可验证、可差异化"，三者缺一则降档。

---

### Dimension 2：Gap Analysis（缺口分析）

区分**硬约束**（无法在申请窗口内解决）和**软劣势**（可以补强的弱点）：

- **硬约束**：重要推荐人拒绝写信、GPA 严重不足（<3.0）、工作授权不满足岗位要求、时间窗口不足 4 周
- **软劣势**：Coding 样本弱、研究叙事碎片化、推荐信组合不优

硬约束 → 触发 Constraint Check（Dimension 5）+ 市场重定向  
软劣势 → 触发 Research Asset Building（见 `content/frameworks/research_asset_building.md`）

---

### Dimension 3：Narrative Coherence（叙事一致性）

**三种叙事状态：**

- **主线型**：所有经历指向同一个核心研究问题，面试官可以一句话复述
- **伪主线型**：表面有主线，细追会发现各段经历间没有真实逻辑连接
- **碎片型**：多个不相关方向并存，面试官无法形成印象

诊断方法及修复路径见 `content/frameworks/research_interest_narrative.md`（含"两连问陷阱"识别和 Reframe as Extension 操作）。

---

### Dimension 4：Positioning（市场定位）

将候选人 profile 映射到以下子市场：

| 子市场 | 典型定位 |
|---|---|
| US Top RA（高竞争） | 强研究经历 + 顶推 + Coding 强 |
| US Mid-tier RA | 研究经历 + 1-2 封有分量推荐信 |
| HK / Asia RA | 对 GPA 容忍度相对高，机构网络重要 |
| European Predoc | 类 PhD 结构，技术门槛高，需独立研究能力 |
| Domestic 科研助理 | 中文发表背景，PI 直接接触为主 |

定位逻辑：先排除硬约束不匹配市场，再按信号强弱排序，最终确定主攻 + 保底分布。

---

### Dimension 5：Constraint Check（约束核查）

| 约束类型 | 核查项 |
|---|---|
| 工作授权 | OPT 剩余时长、STEM Extension 资格、H1B 赞助意愿（见 `visa_constraints`）|
| 地理约束 | 是否受机构位置限制（两地、家庭因素）|
| 时间线 | 可申请的截止日期 vs. 资产补强所需时间 |
| 反向约束 | 导师/学校是否有隐性限制（如禁止在读期间申请）|

---

## 诊断输出

执行五维诊断后，输出三项内容：

**Priority（优先缺口）**  
列出最影响竞争力的 1-2 个缺口，并标注其为硬约束还是软劣势。

**Strategy（补强策略）**  
针对每个优先缺口，指定补强路径（具体执行见 `content/frameworks/research_asset_building.md`）。  
Target market 定位确认。

**Timeline（时间线）**  
补强行动开始日期 → 申请材料截止日期 → 各节点 milestone。  
与 `content/SOP/academic_career_planning_sop.md` 的阶段性节点对齐。

## 典型失败模式

**FM-1：信号误认为成果**  
候选人把"有做过相关研究"认为等同于"有可展示的研究信号"。信号须有具体可叙述内容，否则不计入 Signal Inventory。

**FM-2：软劣势当硬约束处理**  
Coding 弱、叙事碎不是硬约束，是可补强的软劣势。误判导致提前放弃强市场。

**FM-3：跳过 Positioning 直接套策略**  
不先定位目标市场，就直接开始补强材料。结果是为错误的评估维度补强。

**FM-4：时间线倒逼下的"伪诊断"**  
截止日期逼近时跳过诊断直接冲，导致关键缺口未被识别，进入面试后才暴露。

**FM-5：只诊断一次**  
Profile Diagnosis 不是一次性操作。每次出现新信号（新推荐人、新经历、新 offer）或进入新申请季前，应重新执行。
