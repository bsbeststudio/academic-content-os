# Predoc 面试考察逻辑

客观知识。记录面试官为什么问每道题、真正想确认什么。不含个人建议。

---

## 每道题的考察原因

### 研究兴趣与动机
CV 和申请材料几乎都会声称有研究兴趣。面试官需要验证这个兴趣是内生的还是为迎合岗位包装出来的。Predoc 日常工作高度依赖主动性，内在驱动力是实际工作的前提条件。

### 研究兴趣转变
当候选人背景与 lab 研究方向存在明显差距时，面试官需要判断这个转变有真实的学术逻辑，还是为申请临时包装的。这是对研究诚意的直接测试。

### Replication / Coding Exercise 逐步讲解
Coding exercise 是申请的核心材料之一。面试官需要确认候选人真正理解了自己提交的每一行代码——而不是让 AI 或他人做了主要工作后自己简单检查了一遍。这道题直接测试候选人在 lab 里能否独立工作。

### AI 辅助编程
AI 工具在学术研究中已经普遍使用，面试官不是要查有没有用 AI，而是要了解候选人如何使用——是人主导还是 AI 主导。这直接反映独立研究能力。

### Robustness Tests
Robustness test 是实证经济学的核心技能。面试官评估候选人是否理解每个 test 对应什么假设，而不是机械地跑了一个命令。

### 数据可视化与结果解读
数据可视化是 predoc 日常工作的重要组成部分。面试官评估候选人能否从数据做到 story，而不只是描述图长什么样。

### 实验设计
大多数 predoc 岗位在 field experiment lab。即使没有直接实验经验，候选人也需要能理解实验设计的基本逻辑——随机化、控制组、结果变量、威胁。

### 背景匹配度（Why A Good Fit）
当背景 gap 可观察时，面试官评估：候选人是否有自知，是否能说出真正可迁移的技能，是否有针对 gap 的具体学习计划。

### 因果推断与识别策略
因果识别是实证经济学的核心标准。任何声称做实证研究的候选人都必须能清晰说明识别策略——或者诚实承认其不存在。

### 未来计划
PI 需要判断候选人是否会全力投入 predoc 工作，还是把岗位当跳板。投入度直接影响日常工作质量。

## 面试官具体想确认什么

| 题目 | 面试官想确认 |
|---|---|
| 研究兴趣 | 兴趣来自真实的知识积累，而非外部触发 |
| 兴趣转变 | 有内在学术逻辑，而非机会主义 |
| Replication | 候选人控制了所有智识判断 |
| AI 使用 | 人在前，AI 在后 |
| Robustness test | 理解每个 test 对应的假设违反 |
| 可视化 | 能从数据做到研究叙事 |
| 实验设计 | 知道 treatment/control/outcome/threats |
| 背景匹配 | 了解自身差距；有可迁移技能；有学习计划 |
| 因果推断 | 知道用了什么识别策略，或诚实承认没有 |
| 未来计划 | 对岗位有承诺；发展叙事与 lab 方向匹配 |

## 面试结构信息

- 典型时长：30–45 分钟
- 形式：对话式，非结构化问答
- 常见追问方式：面试官对感兴趣的话题会持续深挖
- 答案有效的信号：面试官提出有实质内容的追问
- 答案无效的信号：面试官迅速转题
