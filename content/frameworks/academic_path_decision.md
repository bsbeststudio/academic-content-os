# 学术路径决策框架

## 路径判断逻辑树

```
推荐信主要在北美核心节点？
├── 是 + 研究经历充分 + 签证无障碍
│   └── → 优先直申 PhD
└── 否（国内机构为主 / 经历以课程项目为主）
    └── 是否已在 Master 阶段建立北美 PI 关系？
        ├── 是 → 视推荐信质量决定直申或先 Predoc
        └── 否 → 先 Predoc，通过 Predoc 建立北美推荐信，再申 PhD
```

---

## 签证约束核查（执行路径前必做）

在确认路径前，先回答：
- 目标 Predoc 职位是否要求合法工作资格（OPT/H1B）？
- OPT 剩余时长是否足够完成 Predoc + PhD 申请？
- 如果 OPT 将到期，是否有 STEM 延期资格？

---

## "Why PhD？" 的三层表述结构

面试中被问到职业规划时，按以下三层回答：

**第一层：具体的研究问题意识**
不是"我喜欢研究"，而是"我想理解 X 机制"。给出 1–2 个具体的研究问题。

**第二层：PhD 作为必要路径**
解释为什么回答这个问题必须通过 PhD，而非其他路径。

**第三层：Predoc 在该路径中的具体价值**
说明这个 Predoc 提供了哪些 PhD 申请时需要的能力：实验设计、特定数据技能等。

---

## "Career Goal" 问题的策略处理

**目标定位表述：**
❌ "我想在香港读 PhD"（过窄，传递对市场了解不足）
✅ "我想申请应用微观方向的顶级项目，重点在 [教育/劳动/行为经济学]"

**可以提工业路径，但需呈现为有意图的路径：**
- 说明在 Predoc 阶段会做出明确判断，而非对两条路径同等追求
- 不要让工业路径听起来像是"学术失败的备选"
