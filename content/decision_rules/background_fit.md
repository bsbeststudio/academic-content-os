# 决策规则：背景差距/匹配度问题

## 规则

**如果** 候选人背景与岗位要求之间存在可观察的差距（例如：观测数据背景 vs. field experiment lab），

**则** 不要否认或淡化这个差距。

**而是** 使用三步结构：承认 → 转化为互补性 → 点名学习计划。

## 执行步骤

**第一步 — 直接承认**
> "You're right that I haven't done X before—I want to be upfront about that."

**第二步 — 转化为可迁移优势**
识别 2–3 个真正可迁移的技能：
- 大规模数据处理能力
- 与 lab 研究相关的领域知识
- Coding exercise 中涉及实验数据的部分（如果有）

**第三步 — 点名学习计划**
> "The gap I genuinely have is [X]. I see that as exactly what I should be learning in this role."

## 核心原则

Predoc 本来就是学习岗位。说"我来这里学 X"不是弱点，是对 predoc 定位的正确理解。

## 禁止事项

- 不说"econometric methodology is universal"而没有实质内容跟上。
- 不假装差距不存在。
- 不把还没获得的学习计划描述成已有经验。
